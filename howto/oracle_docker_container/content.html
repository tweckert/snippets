Background

The following instructions describe the steps needed to create a Docker image for Oracle 12.2.0.1 Standard Edition.

Creating an Oracle Technology Network (OTN) user account

  1. Create an Oracle OTN (Oracle Technology Network) user account at https://profile.oracle.com/myprofile/account/create-account.jspx

Creating an Oracle database Docker image

Docker Image vs. Docker Container

An instance of a Docker image is called a Docker container. When you start a Docker image, you have a running Docker container of this image. You can have many running containers of the same image.

Oracle OTN license agreement

Distributing the Oracle database as a "read-to-use" Docker image (e.g. through Docker hub or other Docker repositories) violates the Oracle license agreement. The Oracle OTN license does NOT permit redistribution of the RDBMS binaries at all. Thus, in order to conform to the Oracle license, it is required to download the Oracle binaries from OTN and then do an installation of the Oracle RDBMS in a Docker image with the scripts provided by Oracle as shown below.

Building the Oracle database Docker image

  1. Download the Docker Community Edition from https://www.docker.com/community-edition
    1. For OSX: https://store.docker.com/editions/community/docker-ce-desktop-mac
    2. For Windows: https://store.docker.com/editions/community/docker-ce-desktop-windows
  2. Install Docker and start the Docker host

  3. Clone the Oracle Docker Images github repo at https://github.com/oracle/docker-images

    mkdir oracle-docker-images
cd oracle-docker-images
git init
git remote add origin https://github.com/oracle/docker-images
git fetch
  4. Download from the Oracle OTN the Oracle Database 12c Release 2 (version 12.2.0.1) for Linux x86-64 from http://www.oracle.com/technetwork/database/security/downloads/index.html
    1. Direct download link for Linux x86-64: http://download.oracle.com/otn/linux/oracle12c/122010/linuxx64_12201_database.zip (3.4G)

  5. Move the Oracle Database download into the OracleDatabase/dockerfiles/12.2.0.1 directory of the Oracle github repo checkout for the desired Oracle version:

    Execute these shell commands inside the oracle-docker-images directory
    cd OracleDatabase/dockerfiles/12.2.0.1/
mv ~/Downloads/linuxx64_12201_database.zip .

  6. Build the Docker image with the Oracle database (this will take a few minutes):

    Execute these shell commands inside the oracle-docker-images/OracleDatabase/dockerfiles/ directory
    ./buildDockerImage.sh -v 12.2.0.1 -s

  7. Once building the Docker image has finished, you should see something like this in your console:

    Successfully built 532d2aac656e

  Oracle Database Docker Image for 'se2' version 12.2.0.1 is ready to be extended: 

    --> oracle/database:12.2.0.1-se2
  Build completed in 673 seconds.

Setting up the Oracle database in the Docker container

  1. Execute this command line to start the Docker image "oracle/database:12.2.0.1-se2". Note that "orawac" (Oracle Wacker) is the given name of the Docker container; 1521 is the standard Oracle port number. Again, the first startup of the container will take a few minutes:

    docker run -p 1521:1521 --name orawac oracle/database:12.2.0.1-se2

    On the first startup of the container a random password will be generated for the database. You can find this password in the output line:

    ORACLE PASSWORD FOR SYS, SYSTEM AND PDBADMIN: XXXXXXXXXXXXX

    It is a good idea to keep the Oracle password saved somewhere!

  2. When the database is ready to be used you will see these lines in the output:

    #########################
DATABASE IS READY TO USE!
#########################

  3. On the very first startup of the container a new database is being created. Subsequent startups of the same container just start up the database again. For convenience purposes the container startup script also runs a  tail -f  on the Oracle Database alert.log file. You can see the name of the Oracle database being created in this output line:

    Completed: alter pluggable database ORCLPDB1 open

    The SID of the Oracle instance in the Docker container is ORCLPDB1

    The directory of the Oracle database in the Docker container is /opt/oracle

Configuring hybris to use the Oracle database in the Docker container

Oracle JDBC Thin Driver download

Using the Oracle JDBC Thin Driver for Oracle 12 Release 2 (ojdbc8.jar) causes errors in batch inserts/updates! (For details see: hybris_initialize_w_oracle12.txt)

  1. Download the Oracle JDBC Thin Driver for Oracle 12 Release 1 (ojdb7.jar) from: http://www.oracle.com/technetwork/database/features/jdbc/default-2280470.html
    1. Direct download link: http://download.oracle.com/otn/utilities_drivers/jdbc/121020/ojdbc7.jar
  2. If it's not there already, copy the downloaded JDBC driver JAR (ojdbc7.jar) to wac-hybris/hybris/bin/platform/lib/dbdriver

Creating an Oracle tablespace and user

For security reasons, do not use the SYS, SYSTEM or the SYSMAN database user accounts to run hybris! Create a non-privileged user instead as described below.

  1. Connect to the Oracle database instance "ORCLPDB1" on localhost on port 1521 with the Oracle user SYSTEM and the password that your received during the initial startup of the Oracle database

  2. Create a tablespace (YOURDBTSPACE) and tempspace (YOURDBTEMP):

    SQL
    CREATE TABLESPACE YOURDBTSPACE DATAFILE '/opt/oracle/oradata/yourdbtspace.dbf' SIZE 20M AUTOEXTEND ON LOGGING;
CREATE TEMPORARY TABLESPACE YOURDBTEMP TEMPFILE '/opt/oracle/oradata/yourdbtemp01.dbf' SIZE 5M AUTOEXTEND ON;

    Dropping already existing tablespaces

    SQL
    DROP TABLESPACE YOURDBTSPACE INCLUDING CONTENTS AND DATAFILES;
DROP TABLESPACE YOURDBTEMP INCLUDING CONTENTS AND DATAFILES;

    Selecting all tablespaces

    SELECT TABLESPACE_NAME, STATUS, CONTENTS FROM USER_TABLESPACES;

  3. Create a user (yourdb) with a password (yourdb), and grant permissions to this user:

    SQL
    CREATE USER yourdb IDENTIFIED BY yourdb DEFAULT TABLESPACE YOURDBTSPACE TEMPORARY TABLESPACE YOURDBTEMP PROFILE DEFAULT ACCOUNT UNLOCK;
ALTER USER yourdb DEFAULT ROLE ALL;
GRANT CONNECT TO yourdb;
GRANT RESOURCE TO yourdb;
GRANT CREATE SESSION to yourdb;  
GRANT CREATE TABLE to yourdb;  
GRANT CREATE TRIGGER to yourdb;  
GRANT CREATE VIEW to yourdb;   
GRANT CREATE ANY INDEX TO yourdb;
GRANT UNLIMITED TABLESPACE to yourdb;

    Dropping an already existing user

    SQL
    DROP USER yourdb CASCADE;

    Check database privileges of user 'yourdb'

    SELECT GRANTEE, PRIVILEGE FROM DBA_SYS_PRIVS WHERE GRANTEE = 'YOURDB' ORDER BY PRIVILEGE;
  4. To test the database connection of the yourdb user, use your favourite database management tool (e.g. DbVisualizer). Choose the Oracle Thin Driver and configure the connection as follows:

Configuring the JDBC connection in hybris

  1. Add this JDBC connection to your hybris local.properties file:

    local.properties
    #
# Oracle Thin driver JDBC settings
#
db.url=jdbc:oracle:thin:@//localhost:1521/ORCLPDB1
db.driver=oracle.jdbc.driver.OracleDriver
db.username=yourdb
db.password=yourdb
  2. Afterwards, initialize your hybris application by executing ant initialize. NB: this might take some time (about 30mins)!

Using the Oracle database Docker container

For a complete documentation of all Docker commands, please refer to: https://docs.docker.com/engine/reference/commandline/docker/

The name of the Oracle Docker container is orawac as specified in the "Docker run" command (see this section)

Stopping the Oracle database Docker container

Execute this command line to stop the Docker container of the Oracle database:

docker stop orawac

Starting the Oracle database Docker container

Execute this command line to restart the Docker container of the Oracle database:

docker start orawac

Listing all running Docker containers

Execute this command line to list all running Docker images in your host:

docker ps

Listing all available Docker images

Execute this command line to list all Docker images that are available on your Docker host:

docker images

Logging into the Oracle database Docker container (SSH)

Execute this command line to log into the Oracle database Docker container:

docker exec -t -i orawac /bin/bash

Viewing the logs of the Oracle Docker container

The Docker container runs a process that starts up the Oracle database, and then continues to check each minute if the database is still running, and start it if it's not. Execute this command line to see the output of that process:

docker logs orawac
